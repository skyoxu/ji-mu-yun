## Game Type Specifics
- Game Type: rpg
- Guide Path: docs/game-type-guides/rpg.md
- Character System: Single playable hero with only core combat stats for the prototype: HP and attack. No class split, no leveling tree, no long-term progression.
- World and Exploration: One compact map scene for short exploration with movement, encounter trigger, and clear return flow after battle.
- Combat System: Small-scale RPG combat loop with direct damage exchange, visible HP changes, and explicit win or fail resolution.

### Gameplay Flow / GDD Route
- 使用随机遇怪、地图撞怪，还是二者都支持？ Support both map collision encounter and light random encounter hints, with map collision as the main visible trigger.
- 战斗是回合制指令，还是即时碰撞/自动战斗？ Use a simple turn-based combat command flow with at least Attack as the primary action.
- 胜利后回到地图，还是进入结算后结束 prototype？ After victory, return to the map so the player can immediately feel the loop is connected.
- 玩家失败后是直接 Game Over、Retry 当前战斗，还是回到地图？ After failure, allow retry so the prototype can be validated quickly without restarting the whole project.
- 是否需要战后奖励或肉鸽三选一来验证流派感？ Skip post-battle roguelike reward for the minimum version unless later iteration specifically asks for it.

### Prototype Scene UI
- 战斗场景需要哪些 UI：HP、指令按钮、战斗日志、技能栏？ Battle scene should show player HP, enemy HP, one Attack action, and a short battle log.
- 地图场景需要哪些 UI：HP、任务提示、小地图、遇怪提示？ Map scene should show player HP, movement hint, and a clear encounter or objective hint.
- 失败后是直接 Game Over，还是允许 Retry？ Failure should offer Retry directly.
- 结算 UI 需要哪些按钮：Continue、Retry、Back to Map、End Prototype？ Result UI should provide Continue or Back to Map, plus Retry on failure.
- 是否需要保留调试 UI 帮助快速验证 prototype？ Keep a small debug text area visible if it helps verify encounter and battle state quickly.
